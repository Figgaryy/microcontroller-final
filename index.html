1) Microcontroller คืออะไร?

Microcontroller (MCU) = ชิปคอมพิวเตอร์ขนาดเล็กที่รวม CPU + RAM + Flash + I/O ไว้ในตัวเดียว ใช้ควบคุมอุปกรณ์ขนาดเล็ก เช่น เครื่องซักผ้า, รถยนต์, IoT

ในวิชานี้เราใช้ Arduino UNO ซึ่งใช้ชิป ATmega328P ของบริษัท Atmel/Microchip — มีคุณสมบัติ:

  • F_CPU = 16 MHz (ความเร็ว clock)
  • Flash 32 KB (ที่เก็บโค้ด)
  • SRAM 2 KB (ตัวแปร runtime)
  • EEPROM 1 KB (เก็บข้อมูลถาวร)
  • 14 ขา digital + 6 ขา analog
  • 3 timers (Timer0, 1, 2)
  • 1 USART (Serial port)
  • 1 ADC 10-bit

3) Register, Port, Pin คืออะไร?

Pin (ขา) = สายโลหะออกจากชิป — ใช้เชื่อมกับวงจรภายนอก (LED, สวิตช์, sensor) ATmega328P มีขา I/O 23 ขา

Port (พอร์ต) = กลุ่มของขา 8 ขาที่ควบคุมพร้อมกัน ATmega328P มี 3 ports: PORTB, PORTC, PORTD

  • PORTB มีขา PB0-PB7 (Arduino D8-D13 + crystal pins)
  • PORTC มีขา PC0-PC6 (Arduino A0-A5 + reset)
  • PORTD มีขา PD0-PD7 (Arduino D0-D7)

Register (เรจิสเตอร์) = หน่วยความจำขนาด 8 bits (1 byte) ที่ CPU ใช้ควบคุม hardware การตั้งค่า bit ใน register บอก hardware ว่า "ทำอะไร"

4) Bitwise Operations (ต้องเข้าใจมากที่สุด!) ⭐⭐⭐

การควบคุม bit ใน register เป็นหัวใจของการเขียน MCU ใช้บ่อยมากทั้งใน Q1, Q2, Q3, Q4 ของข้อสอบ

4 operator พื้นฐาน

Opชื่อใช้ทำอะไรกฎ
&ANDเช็คบิต / เคลียร์บิต (ใช้กับ ~mask)1&1=1 อย่างอื่น=0
|ORเซ็ตบิต = 10|0=0 อย่างอื่น=1
^XORToggle บิต (สลับ 0↔1)เหมือนกัน=0 ต่าง=1
~NOTกลับทุก bit (ใช้กับ AND)~0=1 ~1=0
<<Shift Leftเลื่อนซ้าย n ตำแหน่ง1<<3 = 0b00001000
>>Shift Rightเลื่อนขวา n ตำแหน่ง0b11000000 >> 6 = 3

4 patterns ที่จำให้ขึ้นใจ ⭐

Set bit n เป็น 1: x |= (1<<n);

x   = 0b00010000
mask = 0b00000100   // (1<<2)
x|=  = 0b00010100   // bit 2 ขึ้นมา 1

Clear bit n เป็น 0: x &= ~(1<<n);

x         = 0b00010100
mask      = 0b00000100   // (1<<2)
~mask     = 0b11111011   // invert
x &= ~mask = 0b00010000   // bit 2 ลงเป็น 0

Toggle bit n: x ^= (1<<n);

x       = 0b00010100
mask    = 0b00000100   // (1<<2)
x ^= mask = 0b00010000  // bit 2 สลับเป็น 0 (ครั้งหน้าจะเป็น 1)

เช็ค bit n = 1: if (x & (1<<n))

เช็ค bit n = 0: if (~x & (1<<n)) หรือ if (!(x & (1<<n)))

ตัวอย่างจริง: เปิด LED ที่ขา PB5

// 1) ตั้ง PB5 เป็น output
DDRB |= (1<<5);    // DDRB bit 5 = 1

// 2) เปิด LED (HIGH)
PORTB |= (1<<5);   // PORTB bit 5 = 1

// 3) ปิด LED (LOW)
PORTB &= ~(1<<5);  // PORTB bit 5 = 0

// 4) สลับสถานะ LED
PORTB ^= (1<<5);   // PORTB bit 5 toggle

5) Active-low Switch (สวิตช์กด → 0V) ⭐

ใน Lab ทุกสวิตช์เป็น active-low: ปกติอ่านได้ 1 (HIGH), กดแล้วได้ 0 (LOW)

วิธีต่อ: ใช้ pull-up resistor (ภายใน MCU) ดึงขาขึ้น HIGH; เมื่อกดสวิตช์ → ขาต่อ GND → กลายเป็น LOW

DDRD  &= ~(1<<2);   // PD2 = input
PORTD |= (1<<2);    // pull-up enable

// ตรวจปุ่มกด
if (~PIND & (1<<2)) {
    // PD2 = 0 → ปุ่มถูกกด
}
💡 ใน Arduino IDE ใช้ pinMode(2, INPUT_PULLUP) แทนได้ — ตรวจกด: if (digitalRead(2) == LOW)

6) Interrupt คืออะไร? (เปรียบเทียบกับ Polling) ⭐

Polling (เก่า)

CPU ต้อง เช็คเองตลอดว่ามีอะไรเกิดขึ้นไหม → เสียเวลา CPU

while (1) {
    if (TIFR0 & (1<<TOV0)) {
        // timer overflow
        ...
    }
    // ทำอย่างอื่นไม่ได้
}

เปรียบเทียบ: รอบุรุษไปรษณีย์โดยจ้องประตูตลอด

Interrupt (ใหม่ ดีกว่า)

Hardware เรียก CPU ให้มาทำเมื่อเหตุการณ์เกิด → CPU ทำงานอื่นได้

ISR(TIMER0_OVF_vect) {
    // auto-called when overflow
    ...
}
while (1) {
    // ทำอย่างอื่นได้เลย
}

เปรียบเทียบ: ทำงานบ้าน บุรุษไปรษณีย์มาถึงจะกดกริ่งให้

3 ขั้นตอนเปิด Interrupt (ใน C)

  1. ตั้งค่า register ของแต่ละ peripheral (Timer/INT0/PCINT) — เปิด local enable bit
  2. เรียก sei(); เพื่อเปิด global interrupt (set I-flag ใน SREG)
  3. เขียน ISR(VECTOR_NAME) { ... } รับเหตุการณ์

7) Timer / Counter พื้นฐาน ⭐

Timer = ตัวนับ clock อัตโนมัติ — เก็บค่าใน register TCNT0 (8-bit, นับ 0..255)

ทุก clock pulse → TCNT0 เพิ่มขึ้น 1 จนถึง 255 ก็ overflow กลับเป็น 0 แล้ว set flag TOV0

Prescaler = ตัวหารความถี่ ทำให้ Timer นับช้าลง (เพื่อให้ overflow ใช้เวลานานพอที่จะมีประโยชน์)

สูตรหลัก ⭐

F_timer = F_CPU / Prescaler
1 tick = 1 / F_timer

Time delay = (จำนวน tick) × (1 tick)
ticks = Time / (1 tick)

TCNT0 ที่ต้องตั้ง = 256 − ticks      // Normal mode

ตัวอย่าง: ต้องการ overflow ทุก 5 ms ที่ F_CPU 16 MHz

  1. เลือก prescaler 1024 → F_timer = 16,000,000 / 1024 = 15,625 Hz
  2. 1 tick = 1/15,625 = 64 µs
  3. ticks ที่ต้องนับ = 5,000 µs / 64 µs = 78.125 ≈ 78 ticks
  4. TCNT0 = 256 − 78 = 178 = 0xB2

8) PWM (Pulse Width Modulation) คืออะไร? ⭐

PWM = สลับ HIGH/LOW เร็วๆ เพื่อจำลอง "แรงดันกึ่งกลาง" — ใช้คุมความสว่าง LED, ความเร็วมอเตอร์

กำหนด duty cycle = % เวลาที่อยู่ HIGH

  • Duty 0% (value 0) → LED ดับสนิท
  • Duty 25% (value 64) → LED สว่างน้อย
  • Duty 50% (value 128) → LED สว่างกลาง
  • Duty 100% (value 255) → LED สว่างเต็มที่

ใน Arduino ใช้ analogWrite(pin, value) ค่า 0..255 (8-bit) — ใช้ได้เฉพาะ pin ~ 3, 5, 6, 9, 10, 11

// ทำให้ LED ค่อยๆ สว่างขึ้น
for (int v=0; v<=255; v++) {
    analogWrite(9, v);
    delay(10);
}

9) Serial Communication (USART) คืออะไร? ⭐

Serial = ส่งข้อมูลทีละ bit ผ่านสาย 1 เส้น (ต่างจาก parallel ที่ส่งทีเดียว 8 bits)

Arduino ใช้ USART async mode — ส่งทีละ frame:

Start(0) → bit0 → bit1 → ... → bit7 → Stop(1)

Baud rate

= ความเร็วการส่ง bit/วินาที — Lab ใช้ 9600 baud เสมอ

ค่า UBRR0L ที่ใส่ลง register: UBRR = F_CPU/(16×baud) − 1 → ที่ 9600, 16MHz = 103

ขา Serial บน UNO

  • D0 (PD0) = RX รับข้อมูล
  • D1 (PD1) = TX ส่งข้อมูล

10) ADC (Analog-to-Digital Converter) คืออะไร? ⭐

ADC = แปลงสัญญาณ analog (แรงดัน 0-5V ต่อเนื่อง) → ตัวเลขที่คอมพิวเตอร์เข้าใจ

ATmega328P มี ADC ขนาด 10-bit → ค่าที่ออก = 0 ถึง 1023 (= 2^10)

สูตรการแปลง ⭐⭐⭐

ADC = (V_in × 1024) / V_ref
V_in = (ADC × V_ref) / 1024

ที่ V_ref = 5V:
  • 0 V    → ADC = 0
  • 1 V    → ADC = 205      (1×1024/5)
  • 1.5 V  → ADC = 307
  • 2 V    → ADC = 410
  • 2.5 V  → ADC = 512      (กลาง)
  • 3 V    → ADC = 614
  • 3.5 V  → ADC = 717
  • 4 V    → ADC = 819
  • 5 V    → ADC = 1023     (full)
  • 1 LSB ≈ 5/1024 ≈ 4.88 mV

ขา Analog

  • ADC0 = A0 = PC0
  • ADC1 = A1 = PC1
  • ADC2 = A2 = PC2
  • ... ถึง ADC5 = A5 = PC5

วิธีใช้ ADC ใน C

ADCSRA = 0x87;             // ADEN + prescaler 128
ADMUX  = 0x40;             // AVCC ref + ADC0

ADCSRA |= (1<<ADSC);     // start conversion
while (!(ADCSRA & (1<<ADIF)));  // wait done
unsigned int v = ADC;     // อ่านผลลัพธ์ 0..1023

ใน Arduino ใช้ง่ายกว่ามาก: int v = analogRead(A0);

0) Timer คืออะไรในภาษาคนปกติ?

คิดง่ายๆ ว่า Timer = นาฬิกาจับเวลา ในตัวชิป — มันนับเลขขึ้นอัตโนมัติทุกๆ clock pulse (เร็วกว่ามากๆ) จนกว่าจะถึงค่าที่กำหนด

ทำไมต้องมี? เพื่อสร้าง delay ที่แม่นยำ และให้ CPU ทำงานอื่นได้พร้อมกัน

เปรียบเทียบ:

  • วิธีเก่า (loop): "ทำงานอย่างอื่นไม่ได้ ต้องนั่งนับในใจ 1, 2, 3, ..."
  • Timer: "ตั้งนาฬิกาปลุก แล้วไปทำอย่างอื่น พอครบเวลามันจะเตือนเอง"

ส่วนประกอบของ Timer0:

  1. TCNT0 = ตัวเก็บเลข (counter) — นับขึ้นทีละ 1 เรื่อยๆ
  2. TCCR0A, TCCR0B = ปุ่มควบคุม — บอกให้นับยังไง เร็วเท่าไร
  3. TIFR0 = ป้ายแจ้งเตือน (flag) — บอกว่า "นับครบแล้วนะ!"
  4. OCR0A, OCR0B = ตัวเลขเป้าหมาย (ใช้ใน CTC mode)

1) ทำไมต้องใช้ Timer

ก่อนหน้านี้สร้าง delay ด้วย loop เปล่าๆ ซึ่ง:

  • ต้องนับ cycle ของคำสั่งเอง → ผิดพลาดง่าย
  • CPU ติดอยู่ใน loop ทำงานอื่นไม่ได้

Timer/Counter เป็น hardware ในชิปที่นับ clock ให้อัตโนมัติ (ขนานกับ CPU) → CPU ทำงานอื่นต่อได้ และเช็ค flag เมื่อนับครบ

4) Worked Example ⭐ (เหมือน Lab 7 Ex1)

โจทย์: ตั้ง Timer0 normal mode, prescaler 1024 ให้ overflow ทุก 5 ms (5000 µs)

9) CTC Mode (Clear Timer on Compare) ⭐ ใช้ใน Lab 6 Ex2/Ex4

โหมดที่ Timer นับขึ้นเรื่อยๆ จนกระทั่ง TCNT0 == OCR0A ก็จะ:

  • ตั้ง flag OCF0A ใน TIFR0 (bit 1)
  • Reset TCNT0 = 0 อัตโนมัติในรอบถัดไป

การตั้งค่า CTC Mode

Registerค่าหมายเหตุ
TCCR0A0x02WGM01 = 1, WGM00 = 0 → CTC mode
TCCR0B0x04 (pre 256) หรือ 0x05 (pre 1024)WGM02 = 0 + เลือก prescaler
OCR0Aค่า TOP ที่ต้องการเมื่อ TCNT0 ถึง OCR0A → flag OCF0A
TCNT0ค่าเริ่มต้น (มักเริ่มที่ 0)แต่ใน Lab 6 ใช้ค่าเริ่มต้นเฉพาะ

สูตรของ CTC Mode

จำนวน tick = OCR0A − TCNT0_initial
T_delay = (OCR0A − TCNT0_initial) × T_tick
       = (OCR0A − TCNT0_initial) × Prescaler / F_CPU

จาก T_delay ที่ต้องการ →
OCR0A = TCNT0_initial + (T_delay × F_CPU / Prescaler)
TCNT0 = OCR0A − (T_delay × F_CPU / Prescaler)

Worked Example — Lab 6 Ex2

โจทย์: CTC, prescaler 256, TCNT0 = 0x40, ต้องการ delay 2500 µs

  1. F_timer = 16M / 256 = 62,500 Hz → 1 tick = 16 µs
  2. Ticks ใน 2500 µs = 2500 / 16 = 156.25 ≈ 156 ticks
  3. OCR0A = TCNT0 + ticks = 0x40 + 156 = 64 + 156 = 220 = 0xDC
  4. TCCR0A = 0x02 (CTC mode)
  5. TCCR0B = 0x04 (prescaler 256)

การเช็ค flag (Polling)

⚠️ OCF0A อยู่ที่ bit 1 ของ TIFR0 (ไม่ใช่ bit 0). บิต 0 คือ TOV0 (overflow)

การ Stop Timer หลังจบ DELAY

ทำให้ Timer หยุดด้วยการเซ็ต TCCR0B = 0 (ปิด clock):

  • ถ้าเปิด interrupt → flag จะ auto-clearเมื่อเข้า ISR; ถ้า polling ต้อง clear ด้วยมือ

13) คำนวณ Delay ที่นานกว่า Max Timer (Multi-overflow Pattern) ⭐

ปัญหา: Timer0 8-bit + prescaler 1024 → max delay = 16.384 ms; ถ้าอยาก delay 1 วินาทีล่ะ?

วิธีแก้: นับ overflow หลายๆ ครั้ง

// 1 วินาที = 200 × 5 ms
// ตั้ง TCNT0 = 0xB2 → overflow ทุก 5 ms

unsigned char count = 200;
while (count--) {
    // รอ overflow
    while (!(TIFR0 & (1<<TOV0)));
    TIFR0 = (1<<TOV0);              // clear flag
    TCNT0 = 0xB2;                  // reload
}

หรือใช้ ISR + counter (ดีกว่า)

volatile unsigned char z = 200;

ISR(TIMER0_OVF_vect) {
    z--;
    if (z == 0) {
        z = 200;
        PORTB ^= (1<<5);            // toggle ทุก 1 วินาที
    }
    TCNT0 = 0xB2;
}

16) ผิดพลาดที่พบบ่อยกับ Timer ⚠️

  1. ลืม reload TCNT0 ใน ISR → ครั้งต่อไปจะ overflow ที่ 256 ticks (ไม่ใช่ 78) → เวลาผิด
  2. เคลียร์ flag ผิดวิธี — ต้องเขียน 1 ลงไป (ไม่ใช่ 0!) เพื่อ clear: TIFR0 = (1<<TOV0);
  3. ใช้ TCCR0A = 0x01 แทน 0x00 → Fast PWM mode (ไม่ใช่ Normal)
  4. เลือก prescaler 1 หรือไม่ตั้ง prescaler → overflow ทุก 16 µs ค่ะ → CPU โดน interrupt ถี่จัด
  5. ใช้ Timer0 ใน Arduino IDE → delay() / millis() พัง
  6. คำนวณ ticks ผิด เพราะลืมหารด้วย µs ใน 1 tick (ใช้ ms แทน µs ก็พัง)
  7. ลืม sei() หลังตั้ง interrupt → ISR ไม่เรียก ทั้งๆ ที่ flag set

Interrupt = "ขัดจังหวะ" CPU ให้กระโดดไปทำ ISR (Interrupt Service Routine) เมื่อเหตุการณ์เกิด เช่น Timer overflow, ปุ่มกด, ข้อมูลจาก Serial — ไม่ต้องเสียเวลา poll เอง

0) Interrupt คืออะไร — เปรียบเทียบในชีวิตประจำวัน

คิดถึงตอนที่คุณนั่งทำงาน:

  • Polling: เดินไปหน้าประตูทุก 5 นาทีเพื่อเช็คว่ามีไปรษณีย์มาส่งของไหม → เสียเวลา ทำงานอื่นไม่ได้
  • Interrupt: ติดออดที่ประตู ไปรษณีย์มาก็กดออด → คุณทำงานต่อได้สบายๆ พอออดดังค่อยลุกไปรับ

ใน MCU ก็เหมือนกัน: เหตุการณ์ต่างๆ (Timer ครบ, ปุ่มกด, ข้อมูล Serial มา) จะทำให้ CPU หยุดโค้ดปัจจุบัน → กระโดดไป ISR (โค้ดสั้นๆ ที่เขียนไว้) → ทำเสร็จก็กลับมาทำโค้ดเดิมต่อ

ทำไมถึงดี?

  • ตอบสนองเร็ว (ไม่ต้องรอ loop วนมาเช็ค)
  • CPU ทำอย่างอื่นได้ระหว่างรอ
  • ไม่พลาด event แม้ว่าหลายๆ event เกิดเกือบพร้อมกัน (มี priority)

3 ประเภท Interrupt ที่ต้องรู้ในข้อสอบ ⭐

  1. Timer Interrupt — Timer นับครบ → ขัดจังหวะ (ใช้สร้าง periodic event เช่น ทุก 1 วินาที)
  2. External Hardware Interrupt (INT0, INT1) — ขา PD2/PD3 มีสัญญาณเปลี่ยน (rising/falling) → ขัดจังหวะ (ใช้กับปุ่มกด, sensor)
  3. Pin Change Interrupt (PCINT) — ขาใดๆ ในกลุ่ม B/C/D เปลี่ยนสถานะ → ขัดจังหวะ (เหมาะกับสวิตช์หลายตัว)

  3. เขียน ISR วาง ณ address ของ Interrupt vector ที่ต้องการ

7) Important Concepts

9) Interrupt ทำงานอย่างไร — Step by Step ⭐

เมื่อ interrupt เกิดขึ้น CPU ทำตามลำดับ:

  1. เก็บ Program Counter (PC) ปัจจุบันลงใน Stack
  2. Hardware auto-clear I-flag ใน SREG → ป้องกัน nested interrupt
  3. โหลด PC ด้วย address ของ interrupt vector (เช่น 0x0020 สำหรับ Timer0 OVF)
  4. execute JMP T0_OV_ISR ที่อยู่ที่ vector address
  5. กระโดดไปทำคำสั่งใน ISR ทีละบรรทัด
  6. เจอ RETI → restore PC จาก Stack + auto-set I-flag กลับ
  7. กลับไปทำงานเดิมต่อ ราวกับว่าไม่มีอะไรเกิด
BEFORE INTERRUPT:
  PC = 0x0145 (กำลัง execute MAIN loop)
  Stack: [...]
  SREG.I = 1

INTERRUPT EVENT (Timer0 OVF):
  Stack: [..., 0x0145]
  PC = 0x0020 → JMP T0_OV_ISR → PC = 0x0200
  SREG.I = 0 (auto-disabled)

EXECUTING ISR ที่ 0x0200..
  ...
  RETI

AFTER RETI:
  PC = 0x0145 (popped from stack)
  Stack: [...]
  SREG.I = 1 (auto-restored)
  → MAIN loop continues

ทำไมต้อง auto-disable I-flag?

เพื่อป้องกัน nested interrupt — ถ้า ISR หนึ่งกำลังทำงาน อีก interrupt มาขัดอีกที จะ stack overflow ได้ (ATmega328P ไม่ stack guard ในตัว)

ถ้าต้องการให้ interrupt ซ้อนได้จริงๆ ใน ISR ใส่ sei(); ที่จุดที่เห็นว่าปลอดภัย

13) Switch Bouncing — ปัญหาเมื่อต่อกับ INT

ปุ่มกดเชิงกลจะ "bounce" ทำให้ trigger interrupt 5-20 ครั้งต่อการกด 1 ครั้ง

กดจริง:  ───┐                       ┌────
            └─┐ ┐ ┐ ─────┐ ─ ─ ─┘
              └─┘ └─┘ ↑ └─┘
            ↑↑↑↑↑↑    ↑↑↑↑
            falling 5 ครั้ง

วิธีแก้: Software Debounce ใน C

volatile unsigned long last_irq = 0;
volatile uint32_t ms_count = 0;

ISR(TIMER0_OVF_vect) {
    ms_count++;          // 1 ms ticker
    TCNT0 = 131;       // reload
}

ISR(INT0_vect) {
    if (ms_count - last_irq > 50) {     // 50 ms debounce
        last_irq = ms_count;
        PORTB ^= (1<<5);
    }
}

❌ ผิด

unsigned char count = 0;

✅ ถูก

16) Interrupt Common Mistakes ⚠️

  1. ลืม sei(); ใน main → ISR ไม่เคยทำงานทั้งๆ ที่ flag set
  2. ใช้ RET แทน RETI ใน Assembly → I-flag ไม่ถูก restore → interrupt ครั้งต่อไปไม่ทำงาน
  3. ตัวแปร shared ไม่ใส่ volatile → อ่านได้ค่าเดิม (cached)
  4. ทำงานหนักใน ISR เช่น Serial.print, delay → block ระบบทั้งหมด
  5. ลืม clear flag (TOV0) ใน polling → ติดอยู่ในลูปไม่ออก
  6. ลืม pull-up บนปุ่มกด → ขาลอย → trigger interrupt มั่ว
  7. ใช้ EIMSK ผิดบิต เช่น EIMSK = 0x01 สำหรับ INT1 (จริงๆ INT1 = bit 1 = 0x02)
  8. ใช้ PCMSK ผิดกลุ่ม เช่น เปิด PCINT8 (PC0) แต่ใช้ PCMSK0 (กลุ่ม PORTB)
  9. Pin Change ลืมว่า trigger ทั้ง press และ release → counter เพิ่ม 2 ต่อ 1 กด
  10. ใช้ ISR vector name ผิด เช่น TIMER0_VECT (ที่ถูกคือ TIMER0_OVF_vect)

0.5) ADC — เปรียบเทียบกับไม้บรรทัด

ADC แปลงค่า "แรงดันต่อเนื่อง" (analog) ให้เป็น "ตัวเลขกลม" (digital) ที่คอมพิวเตอร์เก็บได้

10-bit ADC = ไม้บรรทัดที่มีขีดวัด 1024 ขีด

  • 0 V = ขีด 0
  • 5 V = ขีด 1023 (เต็มสเกล)
  • 2.5 V = ขีด 512 (กลาง)
  • ขีดละ 5/1024 ≈ 4.88 mV (= ความละเอียดต่ำสุดที่อ่านได้)

สูตรการคิด

ADC value = (V_in × 1024) / V_ref
V_in     = (ADC × V_ref) / 1024

ใน Lab/ข้อสอบใช้ V_ref = 5V (AVCC) เกือบทุกครั้ง

14) Hardware ภายใน USART — เพื่อความเข้าใจ

      [Application]
            ↓ write
      ┌──────────┐
      │ TX Buffer │ ← UDRE0 = 1 ถ้าว่าง
      │  (1 byte) │
      └─────┬────┘
            ↓
      ┌──────────────┐
      │ Shift Reg TX │ → ─── TxD pin (PD1)
      └──────────────┘     (ส่ง bit ทีละตัว)

      RxD pin (PD0) ── → ┌──────────────┐
      (รับ bit ทีละตัว)   │ Shift Reg RX │
                         └──────┬───────┘
                                ↓
                         ┌──────────┐
                         │ RX Buffer │ ← RXC0 = 1 เมื่อมีข้อมูล
                         │  (2 byte) │   (FIFO 2 levels)
                         └──────┬───┘
                                ↓ read UDR0
                          [Application]

Key insight:

  • TX มี 1 byte buffer + shift register → เขียน UDR0 ได้เลยถ้า UDRE0 = 1
  • RX มี 2 byte FIFO → ถ้าอ่านไม่ทัน byte ที่ 3 จะ overrun
  • Hardware ใส่ start/stop bit ให้อัตโนมัติ — ใส่ที่เป็น 8 data bits ก็พอ

15) Library function ที่ดี ✏️

void usart_init(uint16_t ubrr) {
    UBRR0H = (ubrr >> 8) & 0xFF;
    UBRR0L = ubrr & 0xFF;
    UCSR0B = (1<<TXEN0) | (1<<RXEN0);
    UCSR0C = (1<<UCSZ01) | (1<<UCSZ00);     // 8N1
}

void usart_send_char(char c) {
    while (!(UCSR0A & (1<<UDRE0)));
    UDR0 = c;
}

char usart_recv_char(void) {
    while (!(UCSR0A & (1<<RXC0)));
    return UDR0;
}

void usart_send_string(const char* s) {
    while (*s) usart_send_char(*s++);
}

void usart_send_int(int n) {
    if (n < 0) { usart_send_char('-'); n = -n; }
    if (n >= 10) usart_send_int(n/10);
    usart_send_char('0' + (n%10));
}

// usage
int main() {
    usart_init(103);
    usart_send_string("Hello! n=");
    usart_send_int(42);
    usart_send_string("\r\n");
    while(1) {}
}

16) USART RX Interrupt Mode ⭐

แทนที่จะ poll RXC0 ตลอด → ใช้ interrupt ให้ตอบสนองอัตโนมัติเมื่อมีข้อมูล

#include <avr/io.h>
#include <avr/interrupt.h>

volatile char last_ch = 0;
volatile bool got_data = false;

int main() {
    UBRR0L = 103;
    UCSR0B = (1<<TXEN0) | (1<<RXEN0) | (1<<RXCIE0);
    // = 0x10 | 0x08 | 0x80 = 0x98
    UCSR0C = (1<<UCSZ01) | (1<<UCSZ00);
    sei();

    while (1) {
        if (got_data) {
            got_data = false;
            // process last_ch
        }
    }
}

ISR(USART_RX_vect) {
    last_ch = UDR0;        // อ่าน UDR0 → clear RXC0 อัตโนมัติ
    got_data = true;
}
⚠️ ภายใน ISR ต้องอ่าน UDR0 — มิฉะนั้น flag RXC0 ไม่ clear → ISR เรียกซ้ำไม่หยุด

17) ตัวอย่างจริง: รับคำสั่งหลายตัวอักษร

    usart_init(103);
    while (1) {
        char c = usart_recv_char();
        if (c == '\n' || c == '\r') {

        }
    }
}

18) Common USART Mistakes ⚠️

  1. ลืมตั้ง UBRR → baud rate ผิด → terminal แสดงเป็นภาษาต่างดาว
  2. baud rate 2 ฝั่งไม่ตรงกัน → 9600 vs 19200 → garbage data
  3. ใช้ pin D0/D1 เป็น digital I/O ขณะเปิด Serial → Serial พัง
  4. ไม่รอ UDRE0 ก่อนเขียน UDR0 → ส่ง byte ก่อนเขียนทับ
  5. ลืมอ่าน UDR0 ใน ISR(USART_RX_vect) → ISR loop ไม่หยุด
  6. ใช้ printf() ใน AVR-GCC โดยไม่ setup → ต้อง redirect stdio ก่อน (ขั้นสูง)
  7. ไม่ใส่ \r ใน Windows Serial Monitor → บรรทัดต่อกัน

PART B — ADC (10-bit Analog-to-Digital)

1) สเปก ADC ของ ATmega328P

  • ความละเอียด: 10 bit (ค่า 0..1023)
  • Channel: ADC0..ADC5 = pin PC0..PC5 = Arduino A0..A5
  • Vref: ภายนอก (AREF) / AVCC / 1.1V ภายใน
  • ADC clock ที่แนะนำ: 50–200 kHz → ที่ F_CPU 16MHz ต้องใช้ prescaler = 128

2) สูตรแปลงค่า ⭐

ADC = (V_in × 1024) / V_ref
V_in = ADC × V_ref / 1024

ที่ V_ref = 5V:

  • 3.0V → 3.0×1024/5 = 614
  • 5.0V → 1023 (full scale)
  • 1 LSB ≈ 5/1024 ≈ 4.88 mV

Right Justified (ADLAR=0) — ใช้ทั่วไป

ADCH:    [-, -, -, -, -, -, b9, b8]    // 2 bits บน

  3. เริ่มแปลง: ADCSRA |= (1<<ADSC);
  4. รอเสร็จ: while (!(ADCSRA & (1<<ADIF)));
  5. เคลียร์ flag: ADCSRA |= (1<<ADIF); (เขียน 1 เพื่อ clear)
  6. อ่านผล: unsigned int v = ADC;

เคล็ดลับ: เช็ค ADSC แทน ADIF ก็ได้

17) ADC Common Mistakes ⚠️

  1. ลืมตั้ง prescaler → ADC clock เร็วเกิน → ค่าแกว่ง/ผิด
  2. เปิด pull-up บน analog pin → แรงดันบิดเบี้ยว (ห้ามเขียน PORTC |= (1<<n))
  3. ลืมตั้ง REFS → V_ref จะเป็น AREF ที่ floating → ค่าผิด
  4. เปลี่ยน ADMUX แล้วอ่านทันที → ค่าแรกอาจผิด ต้องทิ้งไป 1 ค่า
  5. ใช้ ADC > 5V → ทำลายชิป! ต้อง voltage divider ก่อน
  6. analog pin ลอย (ไม่ต่ออะไร) → ค่ามั่ว ต้อง pull-down หรือต่อ sensor
  7. ลืมรอแปลงเสร็จ → อ่าน ADC ก่อนแปลงเสร็จ → ค่าเก่า

}

ปัญหา: Arduino ขา digital เขียนได้แค่ 0V หรือ 5V (HIGH/LOW) — แล้วถ้าอยาก LED สว่างครึ่งเดียวทำยังไง?

ทางออก: สลับ HIGH/LOW เร็วๆ เกินที่ตามนุษย์จะเห็น (~490 หรือ ~980 Hz) → สมองคนเห็นเป็นความสว่างเฉลี่ย

Duty 100% (value 255):  ████████████  ← HIGH ตลอด → LED สว่างเต็ม
Duty 75%  (value 192):  ████████████░░░  ← HIGH 75% → สว่าง 75%
Duty 50%  (value 128):  ██████░░░░░░  ← HIGH 50% → สว่างครึ่งเดียว
Duty 25%  (value 64):   ███░░░░░░░░░  ← HIGH 25% → สว่างน้อย
Duty 0%   (value 0):    ░░░░░░░░░░░░  ← LOW ตลอด → ดับ

analogWrite(pin, value) ค่า 0..255 → duty cycle 0..100%

2) วงจรพื้นฐาน

LED

  • Anode (ขายาว) → ขา Arduino
  • Cathode (ขาสั้น) → ผ่าน R (220–330 Ω) → GND
  • วิ่งกระแส ~10 mA, ATmega ทนได้ ~20–40 mA/pin

Switch (Active-low)

  • 5V → R 10 kΩ → ขา Arduino → switch → GND
  • ปกติอ่าน HIGH, กดปุ่ม → LOW
  • ใช้ INPUT_PULLUP เปิด resistor ในตัวก็ได้ (ไม่ต้องต่อ R นอก)

Lecture 13 — Arduino: Timer / Interrupt API เสริม

เปรียบเทียบการเขียน Interrupt และ Timer ระหว่าง AVR ดิบ (Lec 7-10) กับ Arduino IDE (Lec 13) — Arduino ห่อทุกอย่างเป็นฟังก์ชันสวยๆ ทำให้เขียนสั้นและ portable

0) ทำไมต้องเรียนเนื้อหาบทนี้?

Lecture 7-10 เราเขียน register โดยตรง (TIMSK0, EIMSK, ...) ซึ่งทำงานได้แต่:

  • ต้องจำชื่อ register, ตำแหน่ง bit เยอะ
  • โค้ดเขียนยาก แก้ลำบาก
  • ติดอยู่กับชิป ATmega328P เท่านั้น

  • attachInterrupt() — ตั้ง external interrupt (INT0, INT1)

  • tone() — สร้างคลื่นสี่เหลี่ยมจาก Timer1/Timer2
  • Library พิเศษ เช่น TimerOne, MsTimer2 สำหรับ periodic interrupt

4) ปัญหาที่พบบ่อย: Switch Bouncing ⭐⭐

เวลากดปุ่ม contact โลหะจะ กระเด้ง 5-50 ms → trigger interrupt 5-20 ครั้งจากการกดครั้งเดียว!

กดจริง: ───┐                     ┌───
            └─┐ ┐ ┐ ┐ ─────┐ ─ ─ ─┘
              └─┘ └─┘     └─┘
            ↑↑↑↑↑↑↑↑    ↑↑↑↑↑
            กดทำเด้ง 8 ครั้ง

วิธีแก้ที่ 1: Software Debounce ใน ISR

volatile unsigned long last_irq = 0;
volatile int counter = 0;

void btnISR() {
    unsigned long now = millis();
    if (now - last_irq > 50) {     // 50 ms debounce
        counter++;
        last_irq = now;
    }
}

ทุกครั้งที่ ISR ทำงานเช็คว่าครั้งที่แล้วผ่าน 50 ms ไหม ถ้ายัง → ignore

6) Non-blocking Pattern ⭐

เป้าหมาย: ทำงานหลายอย่างพร้อมกัน — เช่นกระพริบ LED ทุก 500 ms และอ่าน sensor ทุก 100 ms

❌ ใช้ delay() ทำไม่ได้

void loop() {
    digitalWrite(13, HIGH);
    delay(500);            // CPU block 500 ms
    digitalWrite(13, LOW);
    delay(500);
    // อ่าน sensor ตอนไหน?
}

✅ ใช้ millis() ได้

unsigned long last_blink = 0;
unsigned long last_read  = 0;
int sensor_val = 0;

void loop() {
    unsigned long now = millis();

    // Task 1: Blink LED ทุก 500 ms
    if (now - last_blink >= 500) {
        last_blink = now;
        digitalWrite(13, !digitalRead(13));
    }

    // Task 2: อ่าน sensor ทุก 100 ms
    if (now - last_read >= 100) {
        last_read = now;
        sensor_val = analogRead(A0);
    }

    // ทำงานอื่นได้ทันทีไม่ block
}

10) ISR Best Practices ⭐⭐⭐

กฎทอง

  1. ISR ต้องสั้น เร็ว — ไม่เกิน 100 µs ถ้าเป็นไปได้
  2. ห้ามใช้ delay() ใน ISR — จะ block ทั้งระบบ
  3. หลีกเลี่ยง Serial.print() ใน ISR — ใช้ Timer/buffer ภายใน อาจ deadlock
  4. ตัวแปร shared = volatile — ทุกตัวที่อ่าน/เขียนทั้ง ISR และ loop()
  5. ตัวแปร 16/32-bit ต้อง atomic — อ่าน-เขียนใน loop ต้อง noInterrupts() ครอบ
  6. หลีกเลี่ยง millis() ภายใน ISR ของ Timer0 — เพราะ millis() ใช้ Timer0

Pattern ที่ถูกต้อง: ใช้ Flag

volatile bool got_it = false;

void myISR() {
    got_it = true;          // แค่ตั้ง flag → ออก
}

void loop() {
    if (got_it) {
        got_it = false;
        // งานหนัก เช่น Serial.print, sensor read ทำที่นี่
        Serial.println("Event!");
    }
}

}

    pulses++;
}

📌 Cheat Sheet ที่ควรพกเข้าห้องสอบ (สรุป)

A. Timer formulas

  • F_timer = F_CPU / Prescaler → T_tick = 1 / F_timer
  • Normal: TCNT0 = 256 − ticks (ticks = T_delay × F_timer)
  • CTC: OCR0A = TCNT0 + ticks; TCCR0A = 0x02
  • Prescaler 256 @16MHz → 1 tick = 16µs; 1024 → 64µs

B. Interrupt vectors (C names)

  • TIMER0_OVF_vect, TIMER1_OVF_vect, INT0_vect, INT1_vect
  • PCINT0_vect (PORTB), PCINT1_vect (PORTC), PCINT2_vect (PORTD)
  • USART_RX_vect, ADC_vect

C. Bit ops

  • Set: x |= (1<<n);   Clear: x &= ~(1<<n);   Toggle: x ^= (1<<n);
  • Test bit=1: if (x & (1<<n))   Test=0: if (~x & (1<<n))

D. USART

  • UBRR = F_CPU/(16×baud) − 1 (9600,16M → 103)
  • UCSR0B: TX=0x08, RX=0x10, both=0x18
  • UCSR0C = 0x06 (8N1 async)
  • Wait TX: while(!(UCSR0A&(1<<UDRE0)));
  • Wait RX: while(!(UCSR0A&(1<<RXC0)));

E. ADC

  • ADC = V_in × 1024 / V_ref  |  V_in = ADC × V_ref / 1024
  • ADCSRA = 0x87 (ADEN + prescaler 128)
  • ADMUX: 0x40=AVCC+ADC0, 0x42=AVCC+ADC2, ...
  • Start: ADCSRA |= (1<<ADSC); Wait: while((ADCSRA&(1<<ADIF))==0);

✍️ Written Q1 — Assembly Interrupts (Timer, Pin Change, External)

โจทย์ Q1

เขียนโปรแกรม Assembly สำหรับ ATmega328P (F_CPU = 16 MHz) ที่:

  • มี LED ที่ PB0, PB1, PB2 และสวิตช์ที่ PD2 (INT0), PC0 (PCINT8)
  • Task A (Timer0 OVF): กระพริบ LED1 (PB1) ทุก 1 วินาที — ใช้ Timer0 normal mode, prescaler 1024 (overflow ทุก 5 ms ⇒ นับ 200 รอบ = 1 s)
  • Task B (INT0 falling): เมื่อกดสวิตช์ที่ PD2 → toggle LED0 (PB0)
  • Task C (Pin Change PC0): เมื่อสวิตช์ที่ PC0 เปลี่ยนสถานะ → toggle LED2 (PB2) (เฉพาะตอนปล่อย คือ rising edge — แต่ Pin Change จะ trigger ทั้งสองทาง ให้นับ 2 ครั้ง)

กำหนดค่า register ที่ถูกต้อง พร้อมเขียน main + ISR ทั้ง 3 ตัว

📖 ดูเฉลยละเอียด Q1

ขั้นที่ 3: ค่า register สำหรับ Pin Change PCINT8 (PC0)

  • PCICR = 0x02 (PCIE1 = 1 → group PORTC)

ขั้นที่ 1: เลือก ISR vector C names

ขั้นที่ 2: โค้ด C เต็ม

#include <avr/io.h>
#include <avr/interrupt.h>

volatile unsigned char z = 200;     // 200 × 5ms = 1s

void PIN_SETUP() {
    DDRB  |= (1<<0) | (1<<1) | (1<<2);
    DDRD  &= ~(1<<2); PORTD |= (1<<2);  // PD2 input + pull-up
    DDRC  &= ~(1<<0); PORTC |= (1<<0);  // PC0 input + pull-up
}

void TIMER0_SETUP() {
    TCNT0  = 0xB2;          // 256 − 78
    TCCR0A = 0x00;          // Normal
    TCCR0B = 0x05;          // Prescaler 1024
}

int main() {
    PIN_SETUP();
    TIMER0_SETUP();

    TIMSK0 = (1<<TOIE0);    // 0x01 — Timer0 OVF int enable
    EIMSK  = (1<<INT0);     // 0x01 — INT0 enable
    EICRA  = (1<<ISC01);    // 0x02 — falling edge
    PCICR  = (1<<PCIE1);    // 0x02 — PORTC group
    PCMSK1 = (1<<PCINT8);   // 0x01 — PC0

    sei();                      // global interrupt

    while (1) { }            // main does nothing
    return 0;
}

ISR(TIMER0_OVF_vect) {
    z--;
    if (z == 0) {
        z = 200;
        PORTB ^= (1<<1);     // toggle PB1 every 1s
    }
    TCNT0 = 0xB2;
}

ISR(INT0_vect) {
    PORTB ^= (1<<0);          // toggle PB0
}

ISR(PCINT1_vect) {
    PORTB ^= (1<<2);          // toggle PB2 (press + release)
}
💡 กฎสำคัญของ C ISR: 1) include <avr/interrupt.h> 2) sei() 1 ครั้งใน main 3) ตัวแปรใช้ทั้งใน ISR และ main ต้อง volatile

void usart_send_char(char c) {

    UDR0 = c;

}

void pin_setup() {
    DDRB  |= 0x0F;            // PB0..PB3 output
    PORTB &= ~0x0F;           // LEDs off
    DDRD  &= ~(1<<4);         // PD4 input
    PORTD |= (1<<4);          // pull-up

int main() {
    pin_setup();
    usart_init();
    unsigned char last_btn = 1;

    while (1) {
        // --- TX: ตรวจปุ่ม PD4 ---

        if (last_btn == 1 && btn == 0) {       // falling edge (just pressed)

✍️ Written Q4 — ADC (Register setup + Voltage Mapping)

โจทย์ Q4

เขียนโปรแกรม C สำหรับ ATmega328P (16 MHz):

  • ต่อ potentiometer ไปที่ ADC1 (PC1, A1), V_ref = AVCC = 5V
  • ต่อ LED 4 ดวงที่ PB0..PB3
  • ตามค่าที่อ่านได้:
    • 0 V – 1 V → ติดเฉพาะ LED ที่ PB0
    • 1 V – 2 V → ติดเฉพาะ LED ที่ PB1
    • 2 V – 3.5 V → ติดเฉพาะ LED ที่ PB2
    • มากกว่า 3.5 V → ติดเฉพาะ LED ที่ PB3
📖 ดูเฉลยละเอียด Q4

ขั้นที่ 1: คำนวณ ADC threshold

สูตร: ADC = V × 1024 / V_ref (V_ref = 5V)

แรงดันADC valueคำนวณ
1 V2051 × 1024 / 5 = 204.8 ≈ 205
2 V4102 × 1024 / 5 = 409.6 ≈ 410
3.5 V7173.5 × 1024 / 5 = 716.8 ≈ 717

ขั้นที่ 2: ค่า register

  • ADCSRA = 0x87 (ADEN + prescaler 128 → ADC clk = 125 kHz)
  • ADMUX = 0x41 = AVCC (REFS = 01) + ADC1 (MUX = 0001) + right-justified

ขั้นที่ 3: โค้ดเต็ม

#include <avr/io.h>
#define F_CPU 16000000UL
#include <util/delay.h>

void pin_setup() {
    DDRB  |= 0x0F;             // PB0..PB3 output
    DDRC  &= ~(1<<1);          // PC1 input (analog)
    PORTC &= ~(1<<1);          // no pull-up on analog!
}

void adc_setup() {
    ADCSRA = 0x87;             // ADEN + prescaler 128
    ADMUX  = 0x41;             // AVCC ref + ADC1
}

int main() {
    pin_setup();
    adc_setup();

    while (1) {
        ADCSRA |= (1<<ADSC);                 // start conversion
        while ((ADCSRA & (1<<ADIF)) == 0);   // wait done
        ADCSRA |= (1<<ADIF);                 // clear flag
        unsigned int v = ADC;                  // 0..1023

        PORTB &= ~0x0F;                       // turn off all LEDs first
        if      (v <= 205)              PORTB |= (1<<0);
        else if (v <= 410)              PORTB |= (1<<1);
        else if (v <= 717)              PORTB |= (1<<2);
        else                              PORTB |= (1<<3);

        _delay_ms(100);
    }
    return 0;
}

ขั้นที่ 4: ตรวจคำตอบ Voltage Mapping

ลองเช็ค: ถ้า ADC = 500 → V = 500 × 5 / 1024 ≈ 2.44 V → ควรอยู่ในช่วง 2–3.5 V → LED PB2 ติด ✓

ถ้า ADC = 800 → V = 800 × 5 / 1024 ≈ 3.91 V → > 3.5 V → LED PB3 ติด ✓

⚠️ ข้อผิดพลาดที่พบบ่อย:
1. ลืมตั้ง ADCSRA = 0x87 ทำให้ ADC ไม่ทำงาน
2. ใช้ ADMUX = 0x01 (ลืม REFS = 01) → V_ref จะเป็น AREF (อาจ float)
3. เปรียบเทียบ v <= 1 เพราะลืมแปลง V → ADC value
4. เปิด pull-up บน pin analog → ทำให้แรงดันบิดเบี้ยว

✍️ Written Q1 (ชุดที่ 2) — Assembly: Timer1 + INT1 + PCINT

โจทย์ Q1-2

เขียน Assembly สำหรับ ATmega328P (16 MHz) ที่ทำงานต่อไปนี้พร้อมกัน:

  • Task A: ใช้ Timer0 CTC mode (prescaler 256, OCR0A=0x90, TCNT0=0x00) ให้ toggle PB0 ทุกครั้งที่ match → ออกสัญญาณคลื่นสี่เหลี่ยม

  • Task C: เมื่อขา PB4 (PCINT4) เปลี่ยนสถานะ → toggle PB2

  • TCCR0B = 0x04 (Prescaler 256) — CS02 = 1

    SBI   DDRB, 2           ; PB2 output
    CBI   DDRD, 3           ; PD3 input
    SBI   PORTD, 3          ; pull-up on PD3
    CBI   DDRB, 4           ; PB4 input
    SBI   PORTB, 4          ; pull-up on PB4

    ; --- Timer0 CTC ---
    LDI   R16, 0x00
    OUT   TCNT0, R16
    LDI   R16, 0x90
    OUT   OCR0A, R16
    LDI   R16, 0x02
    OUT   TCCR0A, R16        ; CTC
    LDI   R16, 0x04
    OUT   TCCR0B, R16        ; Prescaler 256
    LDI   R16, 0x02
    STS   TIMSK0, R16        ; OCIE0A enable

    ; --- INT1 rising ---
    LDI   R16, (1<<INT1)
    OUT   EIMSK, R16         ; 0x02
    LDI   R16, (1<<ISC11)|(1<<ISC10)
    STS   EICRA, R16         ; 0x0C

    ; --- Pin Change PB4 ---
    LDI   R16, (1<<PCIE0)
    STS   PCICR, R16         ; 0x01
    LDI   R16, 0x10
    STS   PCMSK0, R16        ; PCINT4

    SEI

LOOP:
    RJMP  LOOP

T0_COMPA_ISR:
    IN    R17, PORTB
    LDI   R18, (1<<0)
    EOR   R17, R18
    OUT   PORTB, R17        ; toggle PB0
    RETI

INT1_ISR:
    IN    R17, PORTB
    LDI   R18, (1<<1)
    EOR   R17, R18
    OUT   PORTB, R17        ; toggle PB1
    RETI

PCINT0_ISR:
    IN    R17, PORTB
    LDI   R18, (1<<2)
    EOR   R17, R18
    OUT   PORTB, R17        ; toggle PB2
    RETI
⚠️ ระวัง: Compare-A vector ของ Timer0 = 0x001C (ไม่ใช่ 0x0020 ที่เป็น OVF). PCINT0 ที่ 0x0006 = group PORTB

✍️ Written Q1 (ชุดที่ 3) — Assembly: นาฬิกานับวินาที

โจทย์ Q1-3

เขียน Assembly เพื่อสร้างนาฬิกาแสดงวินาที:

  • ใช้ Timer0 normal mode + prescaler 1024 → overflow ทุก 5 ms
  • นับ overflow ครบ 200 ครั้ง = 1 วินาที → toggle PB5 (LED ในตัว Arduino)
  • เมื่อกดปุ่ม INT0 (PD2) falling edge → reset counter เริ่มนับใหม่

✍️ Written Q2 (ชุดที่ 2) — C: Timer1 OVF + ADC + LED Bargraph

โจทย์ Q2-2

เขียนโปรแกรม C สำหรับ ATmega328P:

  • ใช้ Timer0 OVF interrupt (prescaler 1024) ให้ กระพริบ LED ที่ PB0 ทุก 0.5 วินาที (= 100 overflows × 5 ms)
  • มี LED bargraph 4 ดวงที่ PB1..PB4
  • เมื่อกดปุ่มที่ PD2 (INT0 falling) → เพิ่มจำนวน LED ที่ติด (1→2→3→4→1...)
  • เมื่อกดปุ่มที่ PD3 (INT1 falling) → ลดจำนวน LED ที่ติด
📖 ดูเฉลยละเอียด Q2-2 
#include <avr/io.h>
#include <avr/interrupt.h>

volatile unsigned char tick = 100;
volatile unsigned char level = 1;     // 1..4

void update_bargraph() {
    PORTB &= ~(0x1E);              // clear PB1..PB4
    for (unsigned char i=0; i<level; i++)
        PORTB |= (1 << (1+i));     // turn on LEDs
}

int main() {
    DDRB |= (1<<0) | 0x1E;        // PB0..PB4 output
    DDRD &= ~((1<<2) | (1<<3));
    PORTD |= (1<<2) | (1<<3);   // pull-up on PD2, PD3

    // Timer0 OVF every 5 ms
    TCNT0  = 0xB2;
    TCCR0A = 0x00;
    TCCR0B = 0x05;
    TIMSK0 = (1<<TOIE0);

    // INT0, INT1 falling edge
    EIMSK = (1<<INT0) | (1<<INT1);   // 0x03
    EICRA = (1<<ISC01) | (1<<ISC11);  // 0x0A

    update_bargraph();
    sei();

    while (1) { }
}

ISR(TIMER0_OVF_vect) {
    tick--;
    if (tick == 0) {
        tick = 100;
        PORTB ^= (1<<0);          // blink PB0
    }
    TCNT0 = 0xB2;
}

ISR(INT0_vect) {
    level++;
    if (level > 4) level = 1;
    update_bargraph();
}

ISR(INT1_vect) {
    level--;
    if (level < 1) level = 4;
    update_bargraph();
}
💡 ตัวแปรที่อ่าน/เขียนทั้งใน main และ ISR ต้องเป็น volatile มิฉะนั้น compiler optimize จะลบ read กลายเป็นค่าผิด

✍️ Written Q2 (ชุดที่ 3) — C: Pin Change + เคลียร์ค่าเดิมก่อน Toggle

โจทย์ Q2-3

เขียน C ให้:

  • มี LED 8 ดวงที่ PORTD ทั้งหมด (PD0..PD7)
  • มีสวิตช์ 3 ตัวที่ PC1, PC3, PC5 (PCINT9, 11, 13)
  • เมื่อสวิตช์ ใดเปลี่ยนสถานะ → shift LED pattern ไปทางซ้าย 1 ตำแหน่ง (LED-rotate)
📖 ดูเฉลยละเอียด Q2-3

คำนวณ PCMSK1

PCINT9 = PC1 = bit 1; PCINT11 = PC3 = bit 3; PCINT13 = PC5 = bit 5

PCMSK1 = (1<<1) | (1<<3) | (1<<5) = 0x02 | 0x08 | 0x20 = 0x2A

#include <avr/io.h>
#include <avr/interrupt.h>

int main() {
    DDRD  = 0xFF;                  // all PORTD output
    PORTD = 0x01;                  // initial: only LED0 on

    DDRC  &= ~0x2A;                 // PC1, PC3, PC5 input
    PORTC |= 0x2A;                  // pull-ups

    PCICR  = (1<<PCIE1);             // 0x02 (PORTC group)
    PCMSK1 = 0x2A;                  // PCINT9, 11, 13

    sei();
    while (1) { }
}

ISR(PCINT1_vect) {
    unsigned char b = PORTD;
    // rotate left 1 bit (with wrap)
    PORTD = (b << 1) | (b >> 7);
}
⚠️ Pin Change trigger ทั้ง press และ release — pattern จะ shift 2 ครั้งต่อการกด 1 ครั้ง. ถ้าต้องการเฉพาะ press ต้องเพิ่ม logic เช็ค PINC ใน ISR

✍️ Written Q3 (ชุดที่ 2) — Serial: เครื่องคิดเลขผ่าน Terminal

โจทย์ Q3-2

เขียน C ให้รับตัวเลข 2 ตัวจาก Terminal (cdar 0-9 ทีละตัว) พร้อมเครื่องหมาย + หรือ - แล้วส่งผลลัพธ์กลับ Terminal:

  • ตัวอย่าง: ผู้ใช้พิมพ์ "5+3" → ตอบกลับ "5+3=8"
  • ผู้ใช้พิมพ์ "9-4" → ตอบกลับ "9-4=5"
  • baud 9600, F_CPU 16 MHz
📖 ดูเฉลยละเอียด Q3-2 
#include <avr/io.h>

void usart_init() {
    UCSR0B = (1<<TXEN0) | (1<<RXEN0);   // 0x18
    UCSR0C = (1<<UCSZ01) | (1<<UCSZ00); // 0x06
    UBRR0L = 103;
}

char usart_recv() {
    while (!(UCSR0A & (1<<RXC0)));
    return UDR0;
}
void usart_send(char c) {
    while (!(UCSR0A & (1<<UDRE0)));
    UDR0 = c;
}

int main() {
    usart_init();
    while (1) {
        char a  = usart_recv();           // '0'..'9'
        char op = usart_recv();           // '+' or '-'
        char b  = usart_recv();           // '0'..'9'

        int n1 = a - '0';
        int n2 = b - '0';
        int r;

        if (op == '+') r = n1 + n2;
        else if (op == '-') r = n1 - n2;
        else continue;

        // echo: a op b = r
        usart_send(a); usart_send(op); usart_send(b);
        usart_send('=');
        if (r < 0) { usart_send('-'); r = -r; }
        if (r >= 10) {
            usart_send('0' + (r/10));
            usart_send('0' + (r%10));
        } else {
            usart_send('0' + r);
        }
        usart_send('\r'); usart_send('\n');
    }
}
💡 เคล็ดลับ: '5' - '0' = 5 เพราะ ASCII '5' = 0x35, '0' = 0x30 ลบกันได้ค่าตัวเลขจริง

✍️ Written Q3 (ชุดที่ 3) — Serial RX Interrupt + LED Display

โจทย์ Q3-3

เขียน C ที่ใช้ RX Complete Interrupt (ไม่ใช่ polling) เพื่อรับตัวเลข ASCII '0'..'9' แล้วแสดงเป็น binary 4-bit บน LED ที่ PB0..PB3:

  • baud 9600, F_CPU 16 MHz
  • เช่น พิมพ์ '7' (= 0x37) → LED แสดง 0111 (PB2=1, PB1=1, PB0=1)
  • ตัวอักษรอื่น → ดับ LED ทั้งหมด
📖 ดูเฉลยละเอียด Q3-3

ขั้นตอนสำคัญ

  1. UCSR0B = TXEN | RXEN | RXCIE → 0x18 | 0x80 = 0x98
  2. เปิด sei(); เพื่อรับ RX interrupt
  3. ใน ISR(USART_RX_vect) อ่าน UDR0 แล้วประมวลผล
#include <avr/io.h>
#include <avr/interrupt.h>

int main() {
    DDRB |= 0x0F;                       // PB0..PB3 output
    PORTB &= ~0x0F;

    UCSR0B = (1<<TXEN0) | (1<<RXEN0) | (1<<RXCIE0);  // 0x98
    UCSR0C = (1<<UCSZ01) | (1<<UCSZ00);
    UBRR0L = 103;

    sei();
    while (1) { }
}

ISR(USART_RX_vect) {
    char c = UDR0;
    PORTB &= ~0x0F;                        // clear LEDs
    if (c >= '0' && c <= '9') {
        PORTB |= (c - '0') & 0x0F;
    }
}
📌 RX Interrupt อ่าน UDR0 ในการแสดง flag RXC0 จะ clear อัตโนมัติ — ไม่ต้องเช็ค flag ใน ISR

✍️ Written Q4 (ชุดที่ 2) — ADC: 5 ระดับแรงดัน

โจทย์ Q4-2

เขียน C สำหรับ ATmega328P (16 MHz):

  • อ่าน potentiometer ที่ ADC3 (PC3, A3), V_ref = AVCC = 5V
  • มี LED 5 ดวงที่ PB0..PB4
  • แบ่ง 5 ช่วง:
    • 0–1V → LED PB0
    • 1–2V → LED PB1
    • 2–3V → LED PB2
    • 3–4V → LED PB3
    • 4–5V → LED PB4
📖 ดูเฉลยละเอียด Q4-2

คำนวณ Threshold

VADCคำนวณ
1V2051×1024/5 = 204.8 ≈ 205
2V4102×1024/5 = 409.6 ≈ 410
3V6143×1024/5 = 614.4 ≈ 614
4V8194×1024/5 = 819.2 ≈ 819

ค่า ADMUX

REFS=01 (AVCC) + ADC3 (MUX = 0011) → 0x43

#include <avr/io.h>
#define F_CPU 16000000UL
#include <util/delay.h>

int main() {
    DDRB |= 0x1F;                   // PB0..PB4 output
    DDRC &= ~(1<<3);
    PORTC &= ~(1<<3);                // no pull-up on analog

    ADCSRA = 0x87;                    // ADEN + prescaler 128
    ADMUX  = 0x43;                    // AVCC + ADC3

    while (1) {
        ADCSRA |= (1<<ADSC);
        while (!(ADCSRA & (1<<ADIF)));
        ADCSRA |= (1<<ADIF);

        unsigned int v = ADC;
        PORTB &= ~0x1F;

        if      (v < 205)  PORTB |= (1<<0);
        else if (v < 410)  PORTB |= (1<<1);
        else if (v < 614)  PORTB |= (1<<2);
        else if (v < 819)  PORTB |= (1<<3);
        else              PORTB |= (1<<4);

        _delay_ms(100);
    }
}

✍️ Written Q4 (ชุดที่ 3) — ADC + Serial Output (Voltmeter)

โจทย์ Q4-3

สร้าง โวลต์มิเตอร์ดิจิทัล:

  • อ่าน ADC0 (PC0, A0) ทุก 500 ms
  • คำนวณแรงดันเป็นทศนิยม 2 ตำแหน่ง
  • ส่งออก Terminal ในรูป "V = X.XX V\r\n"
📖 ดูเฉลยละเอียด Q4-3

คำนวณแรงดัน

V = ADC × 5 / 1024 (เป็น float)

แต่ AVR หลีกเลี่ยง float ใช้ integer math: V × 100 = ADC × 500 / 1024

#include <avr/io.h>
#define F_CPU 16000000UL
#include <util/delay.h>

void usart_init() {
    UCSR0B = (1<<TXEN0);
    UCSR0C = (1<<UCSZ01) | (1<<UCSZ00);
    UBRR0L = 103;
}
void usart_send(char c) {
    while (!(UCSR0A & (1<<UDRE0))); UDR0 = c;
}
void send_str(const char* s) {
    while (*s) usart_send(*s++);
}
void send_int(unsigned int n) {
    char buf[6]; int i = 0;
    if (n == 0) { usart_send('0'); return; }
    while (n) { buf[i++] = '0' + (n % 10); n /= 10; }
    while (i--) usart_send(buf[i]);
}

int main() {
    DDRC &= ~(1<<0);

    ADCSRA = 0x87;
    ADMUX  = 0x40;             // AVCC + ADC0

    usart_init();

    while (1) {
        ADCSRA |= (1<<ADSC);
        while (!(ADCSRA & (1<<ADIF)));
        ADCSRA |= (1<<ADIF);
        unsigned int v100 = ((unsigned long)ADC * 500) / 1024;
        unsigned int whole = v100 / 100;
        unsigned int frac  = v100 % 100;

        send_str("V = ");
        send_int(whole);
        usart_send('.');
        if (frac < 10) usart_send('0');
        send_int(frac);
        send_str(" V\r\n");
        _delay_ms(500);
    }
}
💡 ADC = 614 → v100 = 614×500/1024 = 299 → "V = 2.99 V" ≈ 3.0V (ตรงกับ V_in = 3V)
